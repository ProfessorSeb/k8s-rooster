Key Concepts

Gateway API Resources

Standard Kubernetes resources that kgateway implements:

  • GatewayClass — Defines the controller (kgateway)
  • Gateway — Configures listeners (ports, hostnames, TLS)
  • HTTPRoute — Routes HTTP traffic to backend services
  • TCPRoute — Routes TCP traffic (experimental)

AI Gateway

Specialized protection for AI workloads and LLMs — rate limiting, access control, anomaly detection, and prompt/response filtering for large language models.

Envoy Data Plane

kgateway uses Envoy proxy as its data plane, inheriting dynamic configuration, telemetry, load balancing strategies, circuit breaking, and plugin extensibility.

Multi-Tenancy

Advanced support for multi-tenant environments through namespace isolation, policy scoping, resource quota enforcement, and per-gateway security boundaries.

Example Workflows

1

Install kgateway

> Install kgateway v2.0.1 into the kgateway-system namespace with the Gateway API CRDs.

The agent applies Gateway API CRDs from the official URL, installs kgateway-crds and kgateway Helm charts, verifies pods are running, and confirms the GatewayClass is ready.

2

Expose a Service via HTTPRoute

> Create a Gateway on port 8080 for api.prod.com and route /users to user-service and /orders to order-service.

Generates Gateway + HTTPRoute YAML with path-based routing, applies it, and verifies the routes are programmed by checking Gateway and HTTPRoute status conditions.

3

Troubleshoot Broken Routing

> My HTTPRoute for billing-service isn't routing traffic. What could be wrong?

Inspects the HTTPRoute status, checks parentRef binding to the Gateway, verifies the backend service exists, reads kgateway controller logs, and identifies the root cause.

4

Configure TLS Termination

> Set up TLS termination on the edge-gateway for *.example.com using the wildcard-cert secret.

Updates the Gateway listener to use HTTPS with the TLS secret reference, configures certificate rotation, and verifies the listener is accepting connections.

5

AI Gateway for LLM Protection

> Explain the AI Gateway capabilities and how to protect my LLM endpoint with rate limiting and access control.

Retrieves official documentation on AI Gateway features, explains configuration options for rate limiting, prompt filtering, and token-based access control for LLM endpoints.

Sample Configuration

Gateway + HTTPRoute to expose a service through kgateway:

gateway + httproute
# Gateway resource
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
name: my-http-gateway
namespace: kgateway-system
spec:
gatewayClassName: kgateway
listeners:
- protocol: HTTP
port: 8080
hostname: mydomain.com
name: http
allowedRoutes:
namespaces:
from: All
---
# HTTPRoute resource
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
name: example-route
namespace: example-namespace
spec:
parentRefs:
- name: my-http-gateway
namespace: kgateway-system
hostnames:
- mydomain.com
rules:
- backendRefs:
- name: example-service
port: 80

Installation Commands

kgateway install
# 1. Install Gateway API CRDs
$ kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.2.1/standard-install.yaml
# 2. Install kgateway CRDs
$ helm upgrade -i --create-namespace --namespace kgateway-system \
--version v2.0.1 kgateway-crds \
oci://cr.kgateway.dev/kgateway-dev/charts/kgateway-crds
# 3. Install kgateway
$ helm upgrade -i --namespace kgateway-system \
--version v2.0.1 kgateway \
oci://cr.kgateway.dev/kgateway-dev/charts/kgateway
# 4. Verify installation
$ kubectl get pods -n kgateway-system
$ kubectl get gatewayclass kgateway

Troubleshooting Guide

Symptom Check Fix
HTTPRoute not routing traffic Check status.parents on the HTTPRoute — ensure parentRef matches the Gateway Verify Gateway name/namespace in parentRefs, ensure backend service exists
Gateway not accepting listeners Check status.conditions on the Gateway resource Verify GatewayClass exists, check kgateway controller logs for reconciliation errors
kgateway pods not starting Check pod events and logs in kgateway-system Verify Helm chart version compatibility, check resource limits, ensure CRDs are installed
TLS not terminating Verify the TLS secret exists in the Gateway's namespace Ensure the secret type is kubernetes.io/tls with valid tls.crt and tls.key
Multiple kgateway installs conflicting Check for multiple GatewayClass resources Only one kgateway install per cluster — uninstall duplicates

Important Notes

Single Install Per Cluster

kgateway should only be installed once per cluster. Multiple installations will conflict on GatewayClass reconciliation.

Formerly Gloo Gateway

kgateway was formerly known as Gloo Gateway and is now a CNCF project. Documentation may reference the old name in some places.

GitOps Compatible

All Gateway API resources are declarative Kubernetes CRDs, making them fully compatible with ArgoCD, Flux, and other GitOps workflows.
